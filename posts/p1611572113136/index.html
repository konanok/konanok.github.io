MySQL#

官方网站：

  • MySQL
  • MySQL Docker Hub

安装Docker镜像#

docker pull mysql

启动MySQL容器#

docker run -d \
    --name mysql \
    -e MYSQL_ROOT_PASSWORD=my-secret-pw \
    -p 3306:3306 \
    -v /data/mysql:/var/lib/mysql \
    mysql

上述命令中：

  • -d表示后台运行，--name指定容器名称为mysql。

  • -e是给MySQL传递环境变量。MYSQL_ROOT_PASSWORD是root账户的密码，是必须要设置的，上面的命令中，将root密码设置为了my-secret-pw。

  • -p是端口绑定，将容器3306端口映射到主机的3306端口，用于从外部访问容器。格式：-p 主机端口:容器端口。

  • -v是挂载卷，将容器/var/lib/mysql目录挂载到主机的/data/mysql目录下，用于数据持久化。格式：-v 主机目录:容器目录。

执行完命令之后，MySQL容器就已经启动并运行了，这时就可以使用任意一个数据库连接工具（如Navicat、DataGrip）进行访问了。

这里再说一下在容器内部使用mysql-cli操作数据库的两个方法，这两个方法本质上都是使用docker exec命令在正在运行的容器中运行新命令。

  1. bash

首先进入容器内置的bash：

➜  ~ docker exec -it mysql /bin/bash

root@4b97fc38c74e:/#

此时看到shell的prompt已经变成了root@4b97fc38c74e:/#，说明已经进入了MySQL容器内，使用mysql-cli操作数据库：

mysql -uroot -p

输入刚才启动容器时指定的root账户密码，就可以看到已经进入mysql-cli了：

mysql>
  1. mysql

直接执行容器内置的mysql-cli：

docker exec -it mysql mysql -u root -p

启动或关闭MySQL容器#

执行start或stop命令即可：

docker start/stop mysql/[CONTAINER_ID]

PostgreSQL#

官方网站：

  • PostgreSQL
  • PostgreSQL Docker Hub

安装Docker镜像#

docker pull postgres

启动PostgreSQL镜像#

docker run -d \
    --name postgres \
    -e POSTGRES_USER=root \
    -e POSTGRES_PASSWORD=mysecretpassword \
    -p 5432:5432 \
    -v /data/postgresql:/var/lib/postgresql/data \
    postgres

上述命令的参数与MySQL大致相同，具体信息也可在PostgreSQL的Docker Hub主页上查看。

MongoDB#

官方网站：

  • MongoDB
  • MongoDB Docker Hub

安装Docker镜像#

docker pull mongo

启动MongoDB容器#

docker run -d \
    --name mongo \
    -e MONGO_INITDB_ROOT_USERNAME=root \
    -e MONGO_INITDB_ROOT_PASSWORD=secret \
    -p 27017:27017 \
    -v /data/mongodb:/data/db \
    mongo

Redis#

官方网站：

  • Redis
  • Redis Docker Hub

安装Docker镜像#

docker pull redis

启动Redis容器#

docker run -d \
    --name redis \
    -p 6379:6379 \
    redis \
    --requirepass "yourpassword"

如果想开启Redis的数据持久化，可以使用下面的命令：

docker run -d \
    --name redis \
    -p 6379:6379 \
    -v /data/redis:/data \
    redis \
    --requirepass "yourpassword" \
    --appendonly yes

RabbitMQ#

官方网站：

  • RabbitMQ
  • RabbitMQ Docker Hub

安装Docker镜像#

这里直接安装带management插件的版本，指定镜像标签为management即可：

docker pull rabbitmq:management

启动RabbitMQ容器#

docker run -d \
    --name rabbitmq \
    -e RABBITMQ_DEFAULT_USER=user \
    -e RABBITMQ_DEFAULT_PASS=password \
    -p 5672:5672 \
    -p 15672:15672 \
    rabbitmq:management
